Core Architecture Overview

The agdr_core crate implements the AKI specification in safe Rust. It provides a zero-cost abstraction for cryptographically sealing AI decision records at the moment of inference.

Atomic Sealing BLAKE3 hashing and Ed25519 signing occur within a single kernel boundary
PPP Triplet Binding Purpose, Place, and Person are cryptographically bound to each record
Zero-Copy Design Minimizes memory allocations to achieve sub-microsecond latency
PyO3 Interoperability Seamless integration with Python via the agdr_python extension

Summary: The crate turns an AI decision into a permanent, sealed record instantly. It binds who, where, and why to the action, using high-speed cryptography that doesn't slow down your system.

Atomic Kernel Inference Entry Point

Explain this code
This block shows how to create a new record with the three required context fields (Purpose, Place, Person). The seal() closure runs your model's prediction. Inside this closure, the output is hashed and signed before it is returned to your application. This means the model cannot return a result without it being permanently recorded. 
use agdr_core::{AgDR, Purpose, Place, Person};

// 1. Initialize with PPP Triplet
let record = AgDR::new(
    Purpose::from("High-risk credit evaluation"),
    Place::from("compute-node-alpha-east"),
    Person::from("FOI-7702")
);

// 2. Execute inference with atomic sealing
let sealed = record.seal(|| {
    // Your inference logic here
    model.predict(input_features)
})?;

// 3. Result is immutable and court-admissible
assert_eq!(sealed.root_hash().len(), 32);
assert!(sealed.signature().is_valid());

Summary: You wrap your prediction code in record.seal(|| ...). The library handles the cryptography; you just get a verified, sealed record back.

The Atomic Sealing Process

Summary: Input data enters the kernel. It is combined with Purpose, Place, and Person. The payload is hashed, then combined into a root hash, which is signed. The output is a sealed, immutable record.

Using via PyO3 Python Bindings

Explain this code
This example uses the Python wrapper agdr. You create an AgDR object with the required context, then call .seal() with your prediction logic. The result contains the cryptographic proof. 
# Install the Python extension
pip install agdr-python

# Use in Python
from agdr import AgDR, Purpose, Place, Person

record = AgDR(
    purpose=Purpose("Loan approval decision"),
    place=Place("inference-service-ca-central"),
    person=Person("compliance-officer-4410")
)

sealed = record.seal(lambda: credit_model.predict(applicant_data))
print(f"Record hash: {sealed.root_hash.hex()}")
